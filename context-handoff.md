## 已确认产品决策

- 方案：聊天优先的轻量报销 Agent。
- 形态：企业 IM 机器人 + 简单 Web 管理后台。
- 客户：中小企业。
- 费用：通用日常费用。
- 审批：按金额分级。
- 提交：员工确认后提交。
- 决策体系：财务制度 RAG + 结构化规则 + 人工兜底。

## 当前 MVP

员工上传发票，Agent 识别字段并追问业务信息，结合制度与硬规则进行提交前检查，生成草稿并展示风险与依据。员工确认后提交，进入分级审批和财务复核；Web 后台提供基础配置、审核、查询、导出、统计和审计。

## 重要边界

- RAG 不作最终硬性合规裁决。
- Agent 不得编造业务事实、绕过审批或未经员工确认提交。
- MVP 不包含完整差旅、预算、借款、银企支付、深度 ERP 集成和全自动审批。
- Stage 1 需统一 OCR 指标口径，并确认首发平台、规模、制度样本、安全合规、集成、时间与预算。

## 协作约束

- `CR-001` 已生效：凡需要用户帮助、授权、账号/服务开通、配置或部署，Agent 必须立即说明原因与最小操作步骤，并登记到 `workflow/USER_ACTIONS.md`。
- 未经用户明确批准，不尝试替代账号、额外服务、非必要插件、复杂绕行方案或对同一阻塞重复重试。
- `CR-002` 已生效：当 Agent 无法可靠完成，或用户少量直接操作明显更高效时，必须优先请用户处理；只有用户明确反馈也无法完成后，Agent 才能尝试最小可行替代路径。
- 用户优先机制不得用于转移正常的本地文档、编码、测试、验证和已授权可逆操作。
- `UA-001` 已关闭：Git 作者身份已由用户配置并通过验证。
- `UA-002` 已登记：用户需在 Stage 4 架构与实施计划批准前，将至少一份脱敏真实财务制度放入 Git 忽略的 `.private-input/policies/`；当前不阻塞 Stage 1–3。
